# Security Policy

## Purpose
This repository provides templates and sample artifacts for building hands-on GRC experience. It is intended to be **safe for public sharing**.

To protect people, organizations, and systems, **do not submit or publish sensitive information** in issues, pull requests, discussions, or example files.

---

## What NOT to include (Prohibited Data)
Do not include any of the following in this repo:

### Personal Identifiable Information (PII)
- Real names tied to real organizations
- Personal emails, phone numbers, addresses
- Student/client identifiers

### Organizational Confidential Information
- Internal URLs (drive paths, admin console URLs, intranet links)
- Vendor contracts, invoices, pricing terms, donor records
- Internal diagrams revealing security architecture

### Technical Secrets / Security-Sensitive Data
- Passwords, API keys, tokens, secrets
- Recovery codes, MFA QR codes
- IP addresses, hostnames, MAC addresses
- Device serial numbers, asset tags, barcodes
- Screenshots of production/admin tools without heavy redaction

### Incident Details
- Detailed live incident timelines tied to a real organization
- Unredacted logs from real environments

---

## Allowed Content (Safe Examples)
You *may* include:
- Synthetic sample data (fake names, fake devices, fake org)
- High-level process descriptions
- Templates and checklists
- Redacted screenshots that cannot identify a real org or system
- Counts and summaries (e.g., “12 public links found”) with no identifiers

---

## Responsible Disclosure
If you believe you have found a security issue in the repository content (e.g., accidental sensitive data posted):

1. **Do not** open a public issue with sensitive details.
2. Instead, notify the maintainer privately:
   - **Contact:** [ADD YOUR EMAIL OR PREFERRED CONTACT METHOD]

When reporting, include:
- File path(s)
- What data is exposed
- Recommended remediation (remove/redact/rotate)

---

## Removal Requests
If you are a rights-holder or believe content includes sensitive data:
- Contact the maintainer using the method above and include the file path(s).
- The maintainer will remove or redact content as appropriate.
